Kriterien für das Scratch-Projekt

Funktionen / „Features“
  • Alle Programmier-Karten wurden implementiert *
  • Keine Bugs und unerwünschtes Verhalten
  • Spiel hat ein Ziel (Punkte, Timer, etc.) *
  • Spiel ist interaktiv (man kann den Ablauf beeinflussen)
  • Reset: Man kann direkt mit dem Spiel beginnen
Aussehen / „Style“
  • stimmiger, einheitlicher Look (kann aber auch bewusst chaotisch sein)
  • eigene Grafiken
Optional: Zusätzliche Funktionen /„Bonus Features“
  • Multiplayer: Mehrere Spieler können gleichzeitig etwas tun
  • Level: Es gibt mehrere Level mit veränderten Eigenschaften
  • „Welcome“/Title Screen
  • „Game Over“ Screen
